GPT-5.6上线,Codex图标没了,OpenAI这次是真的把聊天框玩明白了

Jul 10 2026 GPT-5.6 - Codex - Work - Sites - Sol - Terra - Codex - OpenAI - ChatGPT - Luna 13 minutes read (About 2017 words)

今天早上一堆人更新完Codex发现,图标不见了,取而代之的是个新版ChatGPT桌面应用,里面多了Work、Codex、Scheduled、Sites几个入口。

同一天,GPT-5.6系列也正式上线,分成三档:Sol(旗舰)、Terra(均衡)、Luna(便宜量大)。

Codex和ChatGPT其实不是合并了，Codex这个产品没死,在新应用里还是独立模式,继续干代码修改、审查PR这些活,你甚至可以把Codex设成默认启动界面。真正的变化是:Codex从一个单独的App,变成了塞进ChatGPT里的一个引擎。

另外也别急着以为GPT-5.6全面换掉了5.5——日常聊天默认还是5.5 Instant在跑,只有你手动选Medium、High这些推理档位,才会调用到Sol。Terra和Luna基本活在Work、Codex和API里,普通聊天窗口碰不到。

Sol/Terra/Luna这套分级什么意思

太阳系命名法,不只是好听。以前OpenAI是版本号打法,GPT-4→4.5→5,现在改成档位打法:不管底层模型怎么迭代,Sol永远代表最强,Terra永远是性价比,Luna永远是又快又便宜那个。

价格差得也挺明显,按百万Token算:

  • Sol:输入5美元/输出30美元
  • Terra:输入2.5美元/输出15美元
  • Luna:输入1美元/输出6美元

Luna的输出价格只有Sol的五分之一,这意味着以后大批量的分类、抽取、批处理任务,不用再硬着头皮全塞给最贵的模型了。

跑分是真的能打

官方数据,Sol在Agents’ Last Exam(55个专业领域的长周期任务测试)拿了53.6分,比Claude Fable 5高13.1分;中等推理强度下依然领先11.4分,但成本大概只有对方四分之一。

Coding Agent Index上Sol拿80分,比Fable 5高2.8分,输出Token不到一半,时间不到一半,成本低三分之一左右。Terminal-Bench 2.1里Sol是88.8%,开Ultra到91.9%,Fable 5是83.1%。


跑分归跑分,但方向挺清楚:以前旗舰模型基本约等于”更慢更贵更烧钱”,这次OpenAI想证明旗舰也能又快又便宜。对普通用户来说,重点不是又赢了几分,是原来舍不得天天用的顶配能力,现在可能真用得起了。

Ultra不是让模型多想,是直接拉了个四人小队

以前调高推理档位,基本等于让模型自己多琢磨一会儿。Ultra不一样,它默认拉4个Agent同时干活——一个查资料、一个分析、一个执行、一个检查,最后汇总。官方跑分里的Ultra也是4个Agent配置。


好处是能并行的活儿真能并行,复杂任务等待时间短,漏活的概率也低。代价是Token消耗可能不降反升,不少人实测下来一边夸它干活猛,一边吐槽额度掉得肉疼。

所以日常改bug、写脚本用Medium或High基本够了,Ultra留给大重构、跨系统研究这种真正的硬骨头,别把最高档当默认档,不然模型活还没干完,你的额度先没了。

比模型本身更值得关注的,是ChatGPT Work

这次发布真正可能改变工作方式的,其实是Work这个入口。OpenAI给它的定位不是聊天助手,是能跨应用跨文件、连续干活好几个小时的Agent。

它能接Gmail、Google Drive、Slack、Teams、SharePoint、日历、CRM这些,从各处取资料,然后直接产出Excel表格、Word文档、PPT、分析报告,甚至能做交互式网站。桌面端还能读本地文件、开内置浏览器、用Computer Use点鼠标敲键盘挪文件。

这跟以前的ChatGPT差别挺大。以前你让它”根据这些资料写份报告”,它给你的是一段文字,你自己还得排版做成PPT。现在你直接说”从Drive里把上月销售数据拉出来,对比预算,找出异常,更新Excel,再按公司模板做张汇报PPT”,它给你的就是处理好的成品文件。

Sites这个功能容易被忽略,但挺有意思

Work里还带了个Sites,能把数据方案直接做成可交互网站,发链接就能分享或公开访问——数据看板、项目进度页、活动日历、产品原型都行,而且底层数据变了,网站还能跟着自动更新。

对程序员来说这可能就是多了种部署方式,但对不会写代码的人来说完全是另一回事。以前从一个想法到能分享出去的网页,中间隔着设计、前端、服务器、域名、部署一整条链路;现在可能就是一句话的事。

Chat、Work、Codex到底怎么分工

  • Chat:讨论、问问题、改文案、头脑风暴,不涉及大量文件操作的轻活儿。
  • Work:研究、数据分析、做PPT整表格、连办公应用、需要跑几个小时的任务,你说目标,它出成品。
  • Codex:写代码、审PR、重构、跑测试、Git操作,过程会留痕,方便开发者知道模型改了什么、跑了什么命令。

底层能力大量共用,区别主要在界面和交付方式上。

2026 ChatGPT 各套餐额度对比：免费版、Go、Plus、Pro 100/200刀

有没有缺点?肯定有

额度依然是老问题,尤其浏览器操作、多Agent并行、Ultra模式,吃额度很快。实测体验也不完全等于官方跑分,几篇实测文章和评论区比较一致的说法是:规划、代码执行、长任务稳定性进步明显,但中文长文、情绪表达、部分审美这块,未必能完全替代Claude。

最后一点也是最实际的:Agent能操作电脑的权限越大,越需要人盯着。能读文件、登网站、发邮件、改数据,意味着一次误操作影响的不再只是”回答错一句话”。执行可以交给AI,权限和最终拍板,还是得握在人手里。
