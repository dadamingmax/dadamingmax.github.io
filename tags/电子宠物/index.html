#电子宠物

Codex电子宠物保姆级教程：/pet唤醒，/hatch自制宠物，/goal让它盯着任务跑

May 5 2026 Codex>ChatGPT 10 minutes read (About 1517 words)

上周，Codex 悄悄更新了。

大多数人注意到的，是屏幕角落突然多了一只会动的宠物。

宠物是表面。真正藏在里面的，是一套让Agent持续干活的新机制。

今天这篇，帮你把这次Codex更新吃透——/pet怎么玩，宠物怎么自制，/goal到底解决了什么问题。

一、说一下那只宠物：它不是摆设

输入 /pet，Codex桌面端的角落会冒出一只小动物。

可以悬浮、跨App显示。会随着任务状态变换动作。

这就解决了一个真实的痛点：让你知道Codex在跑、有没有卡住、是不是在等我输入？

二、两步唤醒

首先确认你用的是 Codex桌面端App，不是纯 CLI。

在输入框输入：/pet

角落就会出现宠物。再输一次，收起。

或者： Settings ~> Appearance ~> Pets

快捷键党用 Cmd+K / Ctrl+K 打开命令面板，同样可以唤醒或收起。

内置宠物可以直接用，不用任何配置。

三、进阶：用 /hatch 搞一只自己的宠物

不喜欢内置的可以自己造一只。

Codex 支持通过 hatch-pet skill创建自定义宠物。

第一步：安装skill：直接输入$skill-installer hatch-pet

第二步：重载skill：按Cmd/Ctrl + K跑Force Reload Skills

第三步：给出提示词，让它帮你生成，或者通过 image2 生成参考图，让其参考图片生成宠物

四、重头戏：/goal

以前Agent跑一轮就停，你得手动推下一步。

/goal 之后：告诉它目标，它自己持续开，你只管关键节点审查。

从手动挡，变成自动导航。

五、/goal 怎么开启？

手动打开。

编辑配置文件： ~/.codex/config.toml

加入： [features] goals = true

或者直接命令行： codex update codex features enable goals

开启之后，核心命令就这几个：

命令 作用
/goal <目标描述> 创建目标，开始持续执行
/goal pause 暂停，中途检查用
/goal unpause 继续执行
/goal clear 清除目标，重新来

pause/unpause这对命令非常重要。

Agent跑偏了不要直接Ctrl+C粗暴停，先pause，看看跑到哪了，补充上下文，再unpause继续。

七、什么时候用，什么时候不用？

适合 /goal 的场景：

  • 项目重构（改动多、轮次长）
  • 批量修bug（重复劳动，交给它跑）
  • 补测试、写文档（不需要每步确认）
  • 排查复杂问题（让它先探路）

人工介入的场景：

  • 涉及支付、权限、数据删除
  • 改动核心业务逻辑
  • 生产环境部署前

让Codex执行重复劳动，让自己做最后的判断。
