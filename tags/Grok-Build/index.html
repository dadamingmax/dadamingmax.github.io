国内 Grok 注册教程：如何开通 Grok，并升级 SuperGrok 订阅？（2026 年 7 月最新版）

Jul 6 2026 SuperGrok>grok升级订阅>Grok>Grok Build 24 minutes read (About 3532 words)

最近很多国内 AI 用户开始关注 Grok。

一方面，是因为 Claude、ChatGPT 这类工具在账号风控、地区限制、内容限制上让不少人觉得麻烦；

另一方面，Grok 在创意写作、网文构思、漫画分镜、图片生成、视频生成、DeepSearch 搜索等场景里，确实有自己的特点。

如果你只是偶尔体验，免费版 Grok 就能用。 但如果你想长期用 Grok 写文章、做图、生成视频、查资料，或者体验更完整的 Grok 功能，那就需要了解 SuperGrok。

这篇直接讲清楚：

  • 国内怎么注册 Grok？
  • Grok 官网和 X 上的 Grok 有什么区别？
  • 免费 Grok 和 SuperGrok 有什么区别？
  • 国内用户怎么升级 SuperGrok？
  • 升级后不显示 SuperGrok 怎么办？

建议收藏，后面注册和升级时可以直接对照操作。

一、国内可以注册 Grok 吗？

可以。

国内用户注册 Grok，主要需要准备两个东西：

  1. 一个邮箱
  2. 一个能正常访问 Grok 的网络环境

邮箱没有太多限制，常见的 QQ 邮箱、163 邮箱、Outlook、Gmail、自定义域名邮箱，一般都可以用来注册。

Grok 官网入口：

打开官网后，按照页面提示注册账号即可。整体流程和注册其他 AI 工具差不多：填写邮箱、接收验证码、完成登录，然后就可以进入 Grok 使用页面。

二、Grok 官网和 X 上的 Grok 有什么区别？

很多人会把 Grok 官网和 X 上的 Grok 混在一起。

简单说：

grok.com 是 Grok 的独立官网。

x.com 是 X，也就是原 Twitter 平台里的 Grok 入口。

两边都能用 Grok，但账号体系和订阅体系并不完全一样。

如果你平时主要刷 X，经常看海外 AI 动态、发帖、看推文，那么可以考虑在 X 里使用 Grok。

但如果你的目标很明确，就是想专门使用 Grok 聊天、写作、生成图片、生成视频、做 DeepSearch 搜索，那更建议直接使用 grok.com 官网。

原因也很简单：

X 会员是围绕 X 平台设计的，Grok 只是其中一部分权益。 SuperGrok 是围绕 Grok 本身设计的，更适合专门使用 Grok 的用户。

所以选择方式可以直接记住一句话：

经常用 X，就考虑 X 会员。 专门用 Grok，就直接看 SuperGrok。 只是想体验，先用免费版。

三、免费 Grok 和 SuperGrok 有什么区别？

免费版 Grok 适合轻度体验。

你可以用它问问题、写简单内容、试试模型风格，也可以体验部分基础功能。但如果你使用频率比较高，很快就会遇到限制。

免费版常见限制包括：

  • 消息次数有限
  • 高峰期容易变慢
  • 图片和视频生成次数较少
  • 上下文长度较短
  • 部分高级功能不可用
  • 新功能不一定第一时间开放

SuperGrok 则更适合高频用户。

升级 SuperGrok 后，通常可以获得更高的使用额度、更稳定的访问体验、更长的上下文、更完整的图片和视频生成能力，以及部分新功能的优先体验资格。

如果你经常使用下面这些功能，SuperGrok 会更有价值：

写公众号文章/ 写知乎回答/ 写短视频脚本/ 写网文大纲/ 做漫画分镜/ 生成图片素材/ 生成视频素材/ 使用 DeepSearch 查资料/ 进行长对话/ 体验语音和 Companion 功能

不过也要说清楚，Grok 不一定适合所有场景。

四、国内用户为什么升级 SuperGrok 会失败？

国内用户注册 Grok 通常不难，真正麻烦的是支付。

SuperGrok 官方订阅一般需要海外支付方式，比如 Visa、MasterCard 等国际银行卡。很多国内用户没有这类卡，即使有虚拟卡，也可能遇到支付失败、风控拒付、账单地址不匹配等问题。

常见失败原因包括：

  • 银行卡不支持海外订阅
  • 虚拟卡被拒付
  • 账单地址不匹配
  • 支付环境异常
  • 账号触发风控
  • 浏览器缓存导致页面异常

所以国内用户升级 SuperGrok，通常有两种方式：

第一种：自己用海外卡在 Grok 官网订阅。

第二种：通过国内 SuperGrok 升级入口，用账号 ID 完成充值。

如果你有稳定可用的海外卡，可以直接走官方订阅。

如果你没有海外卡，或者多次支付失败，可以考虑账号 ID 直充方式。重点是：尽量选择不需要账号密码的方式。

不要随便把 Grok 账号、邮箱密码、X 密码交给别人。

五、国内如何升级 SuperGrok？账号 ID 直充教程

如果你使用的是账号 ID 充值，一般流程很简单。

第一步：先注册并登录 Grok

先打开 Grok 官网：

grok.com

完成注册并登录账号。

确认你能正常进入 Grok 聊天页面后，再进行下一步。

第二步：找到自己的 Grok 用户 ID

账号 ID 是升级 SuperGrok 时最关键的信息。

现在官方已经没办法通过页面直接查看你的 Grok ID 了。

在有登录的 grok 的账户页面端，访问 grok.com/api/auth/session 账户页面，找到您的账户 ID（你的userid就是你的grok ID）。

第三步：进入 SuperGrok 升级入口

如果你没有海外卡，可以使用国内 SuperGrok 升级入口：cnmgrok.com

选择 SuperGrok 对应套餐，按照页面提示填写信息。

填写购买的充值码和你的 grok id 即可完成充值。

这里再次提醒： 账号 ID 直充一般不需要账号密码。 能不提供密码，就不要提供密码。

第四步：等待 SuperGrok 到账

提交成功后，一般需要等待几分钟。

到账后，重新打开 Grok 页面，如果升级成功，页面一般会显示 SuperGrok 相关状态。

如果页面没有立刻变化，不一定代表失败，可能只是缓存或延迟。

可以尝试下面几种方法：

  • 刷新页面
  • 退出 Grok 后重新登录
  • 换一个浏览器打开
  • 清理浏览器缓存
  • 等待几分钟后再查看
  • 进入账号订阅页面确认状态

六、升级 SuperGrok 后不显示怎么办？

很多人升级后遇到的第一个问题就是：钱付了，为什么页面还是普通 Grok？

这种情况通常有几种原因。

1. 浏览器缓存没刷新

这是最常见的情况。

解决方法：

退出账号 关闭浏览器 重新打开 grok.com 再次登录查看

2. 到账有延迟

有时候系统不是秒到账，可能需要等几分钟。

解决方法：

不要重复提交订单 等待几分钟后刷新页面 进入账号设置查看订阅状态

3. Grok 用户 ID 填错

如果 ID 填错，可能会导致无法正常充值到你的账号。

解决方法：

重新进入账号页面 复制完整 Grok ID 和充值填写的 ID 对比 确认是否漏字符或复制错

4. 登录了另一个账号

有些人同时有 X 账号、Grok 官网账号，甚至多个邮箱账号，容易登录错。

解决方法：

确认当前登录的邮箱 确认当前账号 ID 确认充值时填写的是同一个账号

七、SuperGrok 适合哪些人开？

SuperGrok 不是人人都必须开。

如果你只是偶尔问几个问题，免费版就够了。

如果你每天都要高频使用 Grok，那 SuperGrok 会更合适。

比较适合升级的人群包括：

内容创作者/ 公众号作者/ 知乎运营/ 短视频编导/ 网文作者/ 漫画脚本创作者/ AI 绘图用户/ AI 视频用户/ 海外信息搜索用户/ DeepSearch 重度用户/ 喜欢语音聊天和 Companion 的用户

尤其是做内容的人，Grok 的优势很明显：它更适合发散思路、扩写脑洞、整理热点、生成脚本和做多模态素材。

八、Grok 和 SuperGrok 常见问题

1. 国内注册 Grok 需要手机号吗？

一般情况下，邮箱注册即可。具体是否需要额外验证，要以页面实际提示为准。

2. QQ 邮箱能注册 Grok 吗？

通常可以。QQ 邮箱、163 邮箱、Outlook、Gmail 等常见邮箱都可以尝试。

3. SuperGrok 可以用支付宝或微信支付吗？

官方订阅一般不直接支持国内常用支付方式。没有海外卡的用户，可以考虑国内升级入口（cnmgrok.com）。

4. 升级 SuperGrok 需要提供账号密码吗？

不建议提供账号密码。更推荐使用 Grok 用户 ID 直充方式。

5. X Premium+ 和 SuperGrok 是一回事吗？

不是完全一回事。

X Premium+ 是 X 平台会员，Grok 只是其中一部分权益。 SuperGrok 是 Grok 官网的独立订阅，更适合专门使用 Grok 的用户。

6. SuperGrok 适合写代码吗？

适合

尤其是搭配 Grok Build 后，原型开发和快速迭代效率很高。虽然大型重构略逊于 Claude，但对大多数开发者来说已足够实用且性价比高。

7. SuperGrok 更适合什么？

更适合写作、创意、图片生成、视频生成、DeepSearch、长对话、语音聊天和陪伴类功能。

九、总结：国内用户怎么选择 Grok 和 SuperGrok？

如果你只是想体验 Grok，先用免费版。

如果你是 X 重度用户，可以考虑 X 的会员体系。

如果你主要就是为了使用 Grok 本身，建议直接使用 grok.com，并根据需求升级 SuperGrok。

国内用户升级 SuperGrok，最大问题通常不是注册，而是支付。如果没有海外卡，或者官方支付一直失败，可以考虑账号 ID 直充方式。

最后再总结一句：

Grok 适合创作、搜索和多模态生成。 SuperGrok 适合高频使用 Grok 的用户。 国内用户升级时，优先选择不需要账号密码的方式，更安全也更省事。

买了SuperGrok才发现：这200美元免费额度，没人告诉你怎么用

Jun 17 2026 SuperGrok>额度>Grok>Grok Build 8 minutes read (About 1143 words)

SuperGrok 订阅包含约 $200/月的 xAI API 额度，可访问 grok-imagine-image-quality、grok-imagine-video-1.5-preview、grok-4 等高级模型。但绝大多数用户只用了网页聊天，这部分额度几乎全部浪费。

本文提供两条路径：Grok Build 自动化配置，以及手动逐步操作。

核心原理

xAI 网页端走受限通道，图像和视频配额少且功能受限。通过开源工具 CLIProxyAPI（cliproxy） 将 SuperGrok 登录态转换为本地 OpenAI 兼容 API（http://127.0.0.1:8317/v1），即可在 ComfyUI、Open WebUI 等工具中直接调用订阅内的真实模型额度。

解锁后可访问的模型：

模型 能力
grok-imagine-image-quality 高质量/Pro 图像生成
grok-imagine-video-1.5-preview 更长、更连贯的视频，支持参考图/视频
grok-4 更强的对话与推理

  2. 复制 config.example.yaml 到 config.yaml

  • 点击 xAI 登录区域完成 OAuth 授权

ComfyUI 默认 Grok 节点走 Comfy 云通道。需将节点 API 端点替换为 http://127.0.0.1:8317/v1，API key 填入 management key（local）。可在节点配置中直接修改，或使用 Grok Build 指令自动完成。

常见问题

额度如何消耗？ 消耗的是 SuperGrok 订阅本身包含的 API 额度，非额外赠送。专业工作流单次消耗高于网页聊天，但产出效率和质量同步提升。

OAuth 登录失败？ 检查网络是否能正常访问 x.ai，必要时确保代理支持 OAuth 回调。

后续模型更新如何跟进？ 在 Grok Build 中输入”更新我的 Grok 节点和模型列表”，自动维护。

适合人群

不适合：偶尔生成几张图的轻度用户——网页版已经足够，无需折腾本地环境。

配置完成后，你本地运行的就是一个完整的 xAI API 工作站，而不是受限的网页聊天窗口。这才是 SuperGrok 订阅应有的使用方式。

Composer 2.5 接入 Grok Build：终端 AI 编程助手开始进入“长任务执行”阶段

Jun 2 2026 Composer 2.5>Grok>xAI>Grok Build 13 minutes read (About 1964 words)

最近 xAI 放出了一条很值得关注的更新：

Composer 2.5 is now available inside Grok Build.

翻译就是： Composer 2.5 现在已经可以在 Grok Build 中使用了。

很多人可能会觉得只是一次普通的模型更新。但从 AI 编程工具的发展方向来看，这次更新其实挺关键。

因为它说明 Grok Build 不再只是一个“能在终端里写代码的 AI 工具”，而是开始朝着一个更完整的 Agentic Coding 工作台发展。

一、Composer 2.5 到底是什么？

按照 xAI 官方说法，Composer 2.5 是一个快速、智能度很高的模型，特点是：

擅长处理长时间运行的任务，也能更好地跟随复杂指令。

真正做开发的人，会更关注另外几个问题：

  1. 能不能理解整个项目结构？
  2. 能不能连续执行多个步骤？
  3. 能不能先分析再动手，而不是上来就乱改？
  4. 能不能处理多文件、多模块之间的依赖关系？
  5. 能不能在长任务里保持稳定，不跑偏？

Composer 2.5 这次被接入 Grok Build，重点就是补强这些场景。

不是单纯为了聊天，而是更偏向实际工程任务。

二、Grok Build 是什么？

Grok Build 可以理解为 xAI 推出的终端 AI 编程助手。

不是浏览器里的聊天窗口，也不是单纯的代码补全插件。

直接运行在终端里的 coding agent。

你可以在项目目录里启动它，让它读取项目文件、理解代码结构、分析问题、提出计划，并在你确认后执行修改。

这类工具最大的价值是： 它离真实开发环境更近。

因为开发者平时真正工作的地方，本来就是终端、IDE、Git、项目文件夹和调试日志。

Grok Build 进入终端之后，可以更接近实际开发流程。

四、Composer 2.5 适合用在哪些场景？

不是所有任务都需要切到 Composer 2.5。它更适合处理那些上下文长、步骤多、需要持续理解项目结构的任务。

1. 多文件代码修改

比如登录功能，通常会涉及页面、接口请求、token 存储、路由守卫、用户状态管理和异常提示。 这类任务不是改一个文件就能完成的，如果模型只看局部，很容易改漏。Composer 2.5 更适合先理解整体，再分步骤修改。

2. 项目重构

项目重构通常包括拆分组件、整理目录、抽离工具函数、优化接口层、规范命名和删除重复代码。 这类任务很考验模型的上下文保持能力，Composer 2.5 更适合处理这种长任务。

3. 长链路 Bug 排查

真实项目里的 Bug 往往不是一行代码的问题，可能涉及页面、状态管理、接口、环境变量和构建配置。 Composer 2.5 适合顺着调用链持续分析，先定位问题，再给出修改方案。

4. 文档生成和项目交接

比如生成 README、安装说明、运行步骤、API 文档、目录结构说明和部署文档。 这类任务需要模型理解整个项目，而不是凭空写文档，所以也适合交给 Composer 2.5。

七、Composer 2.5 和普通模型怎么选？

建议是：

如果只是简单问答、小范围代码生成、解释报错，默认模型就够用。

但如果任务满足下面任意一个条件，就可以考虑切到 Composer 2.5：

小任务看速度，大任务看稳定。

Composer 2.5 的价值，主要体现在复杂任务和长任务里。

八、总结

如果你已经在使用 Grok Build，这次 Composer 2.5 值得试一下。

它的意义不是替代所有模型，而是让 Grok Build 多了一个更适合长任务和复杂指令的模型选择。

前几天装 Grok Build，被一个特别隐蔽的问题卡了一晚上。

报错长这样

我本地装好 Grok Build 之后，初始化要跳转 xAI 登录认证。浏览器那边一切正常，但终端这边一直转圈，过一会儿就甩出来这段：error sending request for url https://auth.x.ai/.well-known/openid-configuration

我开始全猜错

请求压根没发出去——这是网络层的问题，跟上面那些一个都不沾边。

最反直觉的一点

根本原因是：

浏览器能上不等于终端能上

Grok Build 使用教程（2026 年最新版）

May 26 2026 Grok Build>Grok>SuperGrok 12 minutes read (About 1756 words)

Grok Build 是 xAI 官方推出的终端 AI 编码代理（Coding Agent CLI），于 2026 年 5 月正式进入早期 Beta 阶段。它把强大的 Grok 模型直接塞进你的终端，能帮你完成从规划、搜索、编码、测试到 Git 提交的全流程工作。

它支持交互式 TUI（全屏、鼠标操作、无闪烁）、Plan Mode（先规划再执行）、并行子代理、无头模式（脚本/自动化）、ACP 协议接入其他工具等特性，是目前最接近“专业软件工程师”的终端 AI 工具之一。

1. 前提条件

  • 订阅要求：SuperGrok 或 X Premium Plus 会员（点击查看文章 : 2026 国内最新 SuperGrok 升级订阅教程）

  • 系统支持：macOS、Linux、Windows（推荐 macOS/Linux 体验最佳）。

  • 推荐环境：在你的项目根目录下运行效果最好（它会自动读取仓库结构、AGENTS.md、技能等）。

2. 安装 Grok Build

macOS / Linux

Windows（PowerShell，管理员权限推荐）

安装完成后，终端输入 grok --version 验证是否成功。

3. 首次启动与登录

  1. 进入你的项目目录：cd /path/to/your/project
  2. 启动 Grok Build：grok

  1. 首次登录： 会自动打开浏览器进行 xAI 账号授权（推荐）。

无浏览器环境可手动设置环境变量：export XAI_API_KEY="xai-你的密钥" （密钥可在 https://console.x.ai/team/default/api-keys 获取）

登录成功后会进入全屏交互式 TUI 界面。

4. TUI 界面操作指南

Grok Build 的 TUI 支持鼠标点击、键盘快捷键，界面清晰无闪烁：

输入框：直接输入自然语言指令（如中文或英文）。

常用命令（在 TUI 内输入）：

命令 功能说明
/help 打开动态帮助面板（推荐新手必用）

/feedback 直接向 xAI 团队提交反馈（Beta 阶段非常有用）

5. Plan Mode（强烈推荐用于复杂任务）

输入复杂需求前，先用 /plan 或启动时加标志进入规划模式。

Grok Build 会先输出完整执行计划（分步骤）。 你可以：

  • 逐条批准（Enter）
  • 评论修改某一步
  • 完全重写计划
  • 批准后，所有文件变更都会以干净的 diff 形式展示，确认无误后再执行。

Plan Mode 的三种常用方法

  1. 快捷键切换（最快，推荐日常使用） :在 TUI 中按 Shift + Tab 循环切换模式，直到状态栏显示 Plan。

  1. 使用 Slash 命令: 输入命令/plan或/plan on后按回车。
  2. 启动时直接进入（适合复杂任务）:grok --plan 或结合项目路径：cd your-project && grok --plan

退出或切换回 Normal 模式

再次按 Shift + Tab 切换到 Normal 模式或输入命令：/plan off
